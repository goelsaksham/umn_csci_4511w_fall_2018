Ascanio Vimercati Sanseverino, Msc Computational Statistics & Machine Learning, University College London (2017)
Answered Apr 1, 2017

GRUs are not a special case of LSTMs and here is an example of something that a GRU can do and that a LSTM cant.

Refer to this great post for an explanation of GRU architectures Understanding LSTM Networks (universally recognised as the best explanation out there).

One thing to look at is that they process the last ouput in a very different way. For the GRU it is easy to repeat the last output, by just saturating the input gate to 0. On the other hand in the LSTM the last ouput contributes to the new output by being processed by a tanh layer, being added to the cell state and then being processed by another tanh layer as part of the cell state. In theory it could be possible for the LSTM to reproduce the last output by tuning the weights of the two tanh layers so that they operate an identy transformation, and by forgetting everything that is in the cell state and replacing it with the last output. Though, such tuning would be valid only for a specifcic value of the output.

Thus the LSTM cannot decide, at some point, (stimulated by the necessary input) to start reproducing the last output. On the other hand this is easily possible for the GRU within the limits of the approximation of the input gate saturating to 0.

Philip Pilgerstorfer, MSc Machine Learning & Statistics, University College London (2017)
Answered Apr 1, 2017

No they are not. I will try to give a precise answer and refer to the notation as in the already mentioned blog post Understanding LSTM Networks by Colah.

We can show that the the LSTM output with gates set to [Math Processing Error]ft=zt ([Math Processing Error]zt is from the GRU), [Math Processing Error]it=1ft, open output-gate [Math Processing Error]ot=1, is equal to the GRU output with open-reset gate [Math Processing Error]rt=1, only if the output-activation function of the LSTM (in Colahs post tanh) is replaced by a linear function. As we need to make additional assumptions on the gates of the GRU, it is not a special case of the LSTM (even when ignoring the linear/tanh assumption).
